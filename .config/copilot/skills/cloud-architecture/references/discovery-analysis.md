# Discovery Analysis

## Analysis Priorities

- Align architecture with business objectives and value drivers.
- Review current architecture, workload characteristics, and operational constraints.
- Capture compliance, data sovereignty, and regulatory requirements.
- Define performance, availability, and security requirements.
- Analyze cost baselines and optimization opportunities.
- Assess team skills, operating model, and change readiness.

## Technical Evaluation

- Inventory infrastructure, applications, and data platforms.
- Map dependencies, integrations, and data flows.
- Establish performance baselines and capacity profiles.
- Evaluate security posture, identity model, and network segmentation.
- Quantify cost breakdown by service, region, and environment.
- Identify technical debt, modernization candidates, and risks.
